# BYOK (Bring Your Own Key) 架构

> 用户自定义 LLM Provider 的完整设计

## 设计原则

1. **API Key 永不离开用户设备**(本地加密存储)
2. **服务端不解密**(仅传递给 LLM Provider)
3. **支持多 Provider 自由切换**
4. **路由策略可配置**(主/备/兜底)

## 加密流程

```
用户输入 API Key (sk-xxx)
        ↓
用户输入主密码 (master password)
        ↓
Argon2id 派生加密密钥 (32 字节, 64MB 内存, 3 轮)
        ↓
AES-256-GCM 加密 API Key
        ↓
存储格式: salt(16) + nonce(12) + ciphertext + tag
        ↓
存储位置:
  - Web: localStorage (前端加密)
  - Tauri: ~/.ripple/keys.db (本地 SQLite)
  - 小程序: 微信云开发(用户自己的云空间)
```

## 调用流程

```
用户发起 LLM 请求
        ↓
前端解密 API Key (用主密码)
        ↓
传递给后端(仅本次会话)
        ↓
后端调用 LLM Provider(直接转发,不存储)
        ↓
返回结果给前端
        ↓
后端立即清除内存中的 Key
```

## 安全保证

- ✓ API Key 永不存数据库
- ✓ API Key 永不进日志
- ✓ 使用 OWASP 推荐的 Argon2id(2025 最新参数)
- ✓ AES-256-GCM 提供认证加密(防篡改)
- ✓ 主密码丢失 = Key 丢失(用户需重新输入)
