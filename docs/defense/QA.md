## 类型 1: 差异化质疑(评委最爱问)

### Q1: 「这个功能元宝 / 元器 / 混元已经有了,你们的差异在哪?」

**回答模板:**

> 评委老师好。元宝、元器、混元是腾讯的优秀产品,但定位不同:
>
> - **元宝**: 通用 C 端 AI 助手,什么都聊
> - **元器**: 让企业/创作者在零代码搭建对话型 Agent
> - **混元**: 大模型基础能力(底层 API)
>
> Ripple 不与他们竞争,而是与他们 **互补**:
>
> 1. **专精「早期信号 + 12 Agent 协作」**:这是元器的"零代码"做不到的,需要工程级编排
> 2. **垂直 KOC 场景的 SOP**:不像通用 Agent 的开放对话
> 3. **底层调用混元 API**:我们是混元的"应用层",不是替代品
>
> 类比:**元器是 WordPress(零代码建站),Ripple 是 Notion(深垂直 SaaS)**——同一生态不同位置。

---

### Q2: 「你的预测准确率多少?有数据吗?」

**回答模板:**

> 这是个好问题。让我先把任务定义清楚:
>
> Ripple 的 Oracle 不是"分类预测"(爆/不爆二选一),而是 **"信号排序 + 置信度估计"**:
> - 输入:种子话题
> - 输出:Top 10 趋势,每条带 0-1 置信度 + 5-14 天窗口期
>
> 评估应该用:
> - **NDCG@10**(排序质量)
> - **Coverage**(信号源数量)
> - **Calibration**(声明置信度 80% 时是否真的 80% 准)
>
> 当前阶段:
> - **9/12 个数据源已接通真实 API**(Polymarket/Manifold/HN/GitHub Trending 等)
> - **3 个数据源(微信指数/百度指数/巨量算数)使用 mock 数据**——这些需要商业采购或人工辅助
> - **回测**正在进行中,大赛阶段优先证明"流程可跑通 + 算法合理",而非生产级准确率
>
> **诚实评估**: 大赛 demo 用 mock 数据展示流程,生产前需 3-6 个月历史数据回测。

### Q3: 「MiniMax 4500 次额度用完后怎么办?」

**回答模板:**

> 我们设计了 **三层降级策略**:
>
> 1. **多 Provider 路由(LiteLLM)**:MiniMax 失败自动切混元,混元失败切 DeepSeek
> 2. **BYOK(用户自带 Key)**:Pro 用户用自己的 API Key,完全不用我们的额度
> 3. **本地模型兜底**:Ollama / LM Studio,无网也能跑
>
> 商业模式上:
> - **Free 用户**: 用我们的额度(限量,每天 3 次 Oracle 扫描)
> - **Pro 用户**: 强烈推荐 BYOK,我们只收订阅费
> - **Enterprise**: 自部署 + 私有模型
>
> 这样 LLM 成本不会随用户量线性爆炸。

### Q4: 「腾讯一键复制你们怎么办?」

**回答模板:**

> 单点功能可以被复制,但有几点很难复制:
>
> 1. **完整产品形态**:腾讯内部要做需要"立项 → 排期 → 跨团队协调"
> 2. **领域 SOP**:5 个 Skills(早期信号/内容包/爆款公式/版权/降 AI 味)是我们花时间打磨的
> 3. **数据飞轮**:用户使用越多,KOC 风格画像越准
> 4. **执行速度**:学生团队迭代快,可以做大公司不愿做的"小而美"
>
> 退一步说,**如果腾讯真的做了,我们的最佳归宿是被收购或被招安**——
> 这其实是大赛的隐含目标:让学生用产品证明能力。

## 类型 2: 技术质疑

### Q5: 「群体智能仿真是不是玄学?」

**回答模板:**

> 评委老师的疑虑非常合理。让我说三点:
>
> 1. **我们没把仿真作为主卖点**——主卖点是 **Oracle 早期信号**(可证伪、可回测)。
>    SimPredictorAgent 是 **附加增强模块**,不是核心。
>
> 2. **仿真的定位是"相对预测",不是"绝对预测"**。我们用它 **比较两个内容方案哪个更可能传播**,
>    而不是预测"会有 100 万播放量"。这是合理的科学方法。
>
> 3. **算法借鉴**:
>    - PopSim(arxiv:2512.02533)— LLM 多智能体社交沙箱
>    - MiroFish + OASIS(CAMEL-AI)— 群体仿真框架
>    - 我们做了简化:200 节点 + 10 轮传播,确保可解释性
>
> **诚实**:仿真不是科学定律,而是 **决策辅助工具**。
> 评委可以选择忽略此模块,Oracle + 多 Agent 协作仍然是完整闭环。

### Q6: 「你们的 Claude Code 架构借鉴是不是过度承诺?」

**回答模板:**

>

### Q7: 「12 个 Agent 是不是过度设计?」

**回答模板:**

> 这个质疑很有意义。让我解释设计理由:
>
> 我们参考 Anthropic《Building Effective Agents》的最佳实践:
> - **Workflow** 优先(确定性流程)
> - **Agent** 仅在必要时使用(开放性任务)
>
> 我们的 12 个不是"全 Agent",而是 **混合架构**:
> - **重 Agent**(自主推理): Oracle / ForumDebate / TopicStrategist / ScriptWriter
> - **轻 Agent**(工具型): TrendScout / MaterialCurator
> - **规则 + LLM 混合**: RiskReviewer / FactChecker
> - **算法 + LLM**: SimPredictor / StyleDecoder
>
> 每个 Agent 都有清晰的**输入/输出契约**,可以独立测试。
> 不是为"显得专业"而拆,而是因为 **职责单一原则**。

### Q8: 「LLM 调用成本是否计算了?」

**回答模板:**

> 是的,我们做了详细成本估算:
>
> **单次完整流水线成本(估算)**:
> - Oracle: 0.5K tokens (轻量信号融合)
> - StyleDecoder: 2K tokens (历史作品 ICL)
> - ForumDebate: 8K tokens (3 Agent × 2 轮)
> - TopicStrategist: 3K tokens
> - ScriptWriter: 8K tokens (6 平台版本)
> - FactChecker + Risk: 4K tokens
> - InsightAnalyst: 1K tokens
> - **总计约 26K tokens / 次**
>
> 按 MiniMax M2.7 价格(¥0.50/¥2.00 per 1M):
> - 输入:13K * 0.50 / 1M ≈ ¥0.0065
> - 输出:13K * 2.00 / 1M ≈ ¥0.026
> - **单次 ≈ ¥0.03**
>
> Pro 用户 ¥99/月,假设月用 100 次 = ¥3 成本,**毛利率 97%**。
> 重度用户用 BYOK,我们只收订阅费,毛利更高。

### Q15: 「AI 生成内容合规怎么办?」

**回答模板:**

> 我们提前对齐了 2025-09-01 施行的《AI 生成合成内容标识办法》:

> - 政治敏感词检测

---

### Q16: 「用户的 API Key 安全吗?」

**回答模板:**

> 这是我们最重视的安全设计——**BYOK 本地加密**:
>
> - **Argon2id** 派生密钥(2025 OWASP 推荐参数)
> - **AES-256-GCM** 认证加密
> - **永不上传服务器**:Web 用 localStorage,Tauri 用 macOS Keychain / Windows DPAPI
> - **解密只在调用 LLM 时临时进行**,内存即时清除
> - **用户主密码**:用户自己掌握,丢失 = Key 失效
>
> 防御 OWASP LLM Top 10 v2 中的 LLM02(信息泄露)与 LLM10(无界消耗)。

---

### Q17: 「视频号没有发布 API,你们怎么发?」

**回答模板:**

> 评委说得对,视频号目前没有面向普通创作者的发布 API。

> - **小程序挂载**:视频号挂小程序(归因 wxFinderId)

## 通用应对原则

### 当评委质疑时

1. **认可 + 解释**:先说"这是个好问题",不要否认
2. **分层回答**:短期/中期/长期
3. **诚实 > 夸大**:比起"无敌"叙事,评委更信任"清醒"
4. **用数据 + 具体例子**:避免空洞形容词
5. **5 句话内说清**:不要超过 1 分钟

### 紧急情况

- **现场 Demo 挂了**: 切换到 Streamlit + 截图 + 录屏
- **被问到不会的**: "这个我需要回去研究,可以邮件回复您"
- **被质疑诚信**: "您说得对,我们再想想"

### 自信展示

- ✅ 12 Agent 完整代码可演示
- ✅ Claude Code 架构借鉴可对照
- ✅ STRIDE 威胁建模文档可展示
- ✅ Skills 5 个 SKILL.md 可展示
- ✅ PDF 与 Demo 一致

> **记住**: 评委不是来"找茬"的,是来"找潜力"的。
> 自信、诚实、专业 — 这是赢的关键。
